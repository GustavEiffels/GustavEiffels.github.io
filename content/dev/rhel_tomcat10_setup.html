이관 파일 준비

외부망 PC에서 Apache Tomcat 공식 배포 tar.gz를 다운로드하여 폐쇄망 서버로 이관합니다.

이관 파일: apache-tomcat-10.1.34.tar.gz + ROOT.war (배포 파일)

사전 요건 — Java 17 설치 및 JAVA_HOME 환경변수가 설정되어 있어야 합니다.
java -version으로 확인 후 진행하세요.
2
압축 해제 및 위치 지정

root 권한으로 지정된 서비스 경로에 압축을 풉니다.

root@rhel-test-server 
# 서비스 디렉토리 생성
[root]# mkdir -p /home/test/test

# 지정 경로에 압축 해제
[root]# tar -xvf apache-tomcat-10.1.34.tar.gz -C /home/test/test

# 확인
[root]# ls /home/test/test/apache-tomcat-10.1.34/

1
wget으로 Tomcat 다운로드

Apache 공식 아카이브에서 Tomcat 10.1.34를 직접 다운로드합니다.

root@server 
# 작업 디렉토리
[root]# cd /tmp

# Tomcat 10.1.34 다운로드
[root]# wget https://archive.apache.org/dist/tomcat/tomcat-10/v10.1.34/bin/apache-tomcat-10.1.34.tar.gz

# 서비스 디렉토리 생성 및 압축 해제
[root]# mkdir -p /home/test/test
[root]# tar -xvf apache-tomcat-10.1.34.tar.gz -C /home/test/test
wget이 없으면: dnf install -y wget
공통 설정 및 기동
C1
Connector 최적화 (server.xml)

네트워크 지연 및 대용량 패킷 처리를 위해 maxSwallowSize="-1"을 Connector에 추가합니다.

server.xml 
[root]# vi /home/test/test/apache-tomcat-10.1.34/conf/server.xml

<!-- Connector 섹션에 maxSwallowSize="-1" 추가 -->
<Connector port="8080" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443"
           maxParameterCount="1000"
           maxSwallowSize="-1" />
maxSwallowSize="-1" — 클라이언트 대용량 요청 바디를 끊지 않고 전부 수신합니다. 파일 업로드 오류 방지에 필수입니다.
C2
JVM 메모리 및 프로파일 설정 (setenv.sh)

Tomcat 기동 시 자동으로 로드되는 setenv.sh에 JVM 옵션과 Spring 프로파일을 설정합니다.

setenv.sh 생성 
[root]# echo '#!/bin/sh' > /home/test/test/apache-tomcat-10.1.34/bin/setenv.sh
[root]# echo 'export CATALINA_OPTS="$CATALINA_OPTS -Dspring.profiles.active=prod-deploy"' \
    >> /home/test/test/apache-tomcat-10.1.34/bin/setenv.sh
[root]# echo 'export CATALINA_OPTS="$CATALINA_OPTS -Djava.net.preferIPv4Stack=true"' \
    >> /home/test/test/apache-tomcat-10.1.34/bin/setenv.sh
[root]# echo 'export CATALINA_OPTS="$CATALINA_OPTS -Xms6G -Xmx6G"' \
    >> /home/test/test/apache-tomcat-10.1.34/bin/setenv.sh

# 실행 권한 부여
[root]# chmod +x /home/test/test/apache-tomcat-10.1.34/bin/setenv.sh
⚠️ -Xms6G -Xmx6G는 서버 물리 메모리에 맞게 조정하세요. (일반적으로 물리 메모리의 50~75%)
C3
WAR 배포 및 서버 기동·모니터링

배포 파일(ROOT.war)을 webapps에 이동하고 서버를 기동하여 정상 여부를 확인합니다.

# WAR 파일 배포
[root]# mv ROOT.war /home/test/test/apache-tomcat-10.1.34/webapps/

# 서버 기동
[root]# /home/test/test/apache-tomcat-10.1.34/bin/startup.sh

# 실시간 로그 확인
[root]# tail -f /home/test/test/apache-tomcat-10.1.34/logs/catalina.out
✅ 정상 기동 확인 — 로그에서 Server startup in [N] milliseconds 메시지 확인
브라우저: http://서버IP:8080 접속
서버 관리 명령
중지: /home/test/test/apache-tomcat-10.1.34/bin/shutdown.sh
재시작: shutdown.sh → startup.sh 순서로 실행
