postgre_LIBs.tar.gz 압축 해제 및 RPM 설치

폐쇄망 서버로 이관된 postgre_LIBs.tar.gz (177개 RPM, 약 81MB)를 압축 해제한 뒤 일괄 설치합니다.

이관 파일: postgre_LIBs.tar.gz + postgresql-17.2.tar.gz

root@server — 파일 이관 후 
# 작업 디렉토리 생성 및 이동
mkdir -p /opt/install && cd /opt/install

# 압축 해제
tar -xzf postgre_LIBs.tar.gz

# RPM 파일 수 확인
ls *.rpm | wc -l

# 전체 RPM 일괄 설치
rpm -ivh *.rpm --nodeps --force
✅ 설치 완료 기준 — 177개 RPM 중 오류 없이 진행되면 정상. 이미 설치된 패키지는 --force로 덮어씁니다.

빌드 환경 검증
# GCC 확인
gcc --version

# Perl 핵심 모듈 확인
perl -e "use Getopt::Long; use Exporter; print 'Perl OK\n'"
두 명령 모두 정상 출력이 나와야 컴파일을 진행합니다.
gcc (GCC) 8.5.0 ...  /  Perl OK
3
소스 압축 해제 → Configure → 컴파일 → 설치

--without-icu 옵션으로 ICU 의존성을 제거합니다. 폐쇄망에서 icu 패키지를 별도 수집하지 않은 경우 필수입니다.

postgresql-17.2/ 
cd /opt/install
tar -xzf postgresql-17.2.tar.gz
cd postgresql-17.2

# 빌드 설정 (ICU 제외)
./configure --prefix=/opt/pgsql --without-icu

# 병렬 컴파일
make -j$(nproc)

# 설치
make install

# 설치 확인
/opt/pgsql/bin/postgres --version
✅ 성공 출력 — postgres (PostgreSQL) 17.2
4
환경변수 설정
cat >> ~/.bash_profile <<'EOF'
export PATH=/opt/pgsql/bin:$PATH
export LD_LIBRARY_PATH=/opt/pgsql/lib:$LD_LIBRARY_PATH
EOF

source ~/.bash_profile
5
postgres OS 유저 생성 및 디렉토리 권한 설정

initdb는 root로 실행 불가합니다. 전용 OS 유저를 생성하고 소유권을 부여합니다.

root 권한으로 실행 
useradd -m postgres
mkdir -p /opt/pgsql/data
chown -R postgres:postgres /opt/pgsql
6
DB 초기화 · 서버 시작 · 계정 암호 설정

이하 모든 명령은 postgres 유저로 실행합니다.

postgres 유저로 전환 
su - postgres

# DB 클러스터 초기화
/opt/pgsql/bin/initdb -D /opt/pgsql/data

# 외부 접속 허용
sed -i "s/#listen_addresses = 'localhost'/listen_addresses = '*'/" \
    /opt/pgsql/data/postgresql.conf
echo "host all all 0.0.0.0/0 md5" >> /opt/pgsql/data/pg_hba.conf

# 서버 시작
/opt/pgsql/bin/pg_ctl -D /opt/pgsql/data \
    -l /opt/pgsql/data/postgresql.log start

# psql 접속 후 암호 설정
/opt/pgsql/bin/psql -U postgres
ALTER USER postgres WITH PASSWORD 'your_password';
\q
서버 관리 명령 (postgres 유저)
중지: /opt/pgsql/bin/pg_ctl -D /opt/pgsql/data stop
재시작: /opt/pgsql/bin/pg_ctl -D /opt/pgsql/data restart
로그: tail -f /opt/pgsql/data/postgresql.log

1
PGDG 공식 리포지토리 등록

PostgreSQL 공식 PGDG 리포지토리를 등록합니다. RHEL 8 AppStream 내장 모듈과 충돌하지 않도록 비활성화가 필수입니다.

root@server 
# PGDG 리포지토리 RPM 설치
dnf install -y \
  https://download.postgresql.org/pub/repos/yum/reporpms/EL-8-x86_64/pgdg-redhat-repo-latest.noarch.rpm

# AppStream 내장 PostgreSQL 모듈 비활성화 (버전 충돌 방지)
dnf -qy module disable postgresql
⚠️ AppStream 모듈 비활성화를 생략하면 기본 버전(10.x)과 충돌이 발생합니다.
2
PostgreSQL 17 설치
# 서버 + 클라이언트 설치
dnf install -y postgresql17-server postgresql17

# 설치 확인
/usr/pgsql-17/bin/postgres --version
✅ 성공 출력 — postgres (PostgreSQL) 17.x
3
DB 초기화 → 외부 접속 설정 → 서비스 시작
# DB 클러스터 초기화
/usr/pgsql-17/bin/postgresql-17-setup initdb

# 외부 접속 허용
sed -i "s/#listen_addresses = 'localhost'/listen_addresses = '*'/" \
    /var/lib/pgsql/17/data/postgresql.conf
echo "host all all 0.0.0.0/0 md5" >> /var/lib/pgsql/17/data/pg_hba.conf

# 부팅 시 자동 시작 + 즉시 시작
systemctl enable --now postgresql-17
systemctl status postgresql-17
postgres 계정 암호 설정 
sudo -u postgres psql

ALTER USER postgres WITH PASSWORD 'your_password';
\q
서비스 관리 명령
중지: systemctl stop postgresql-17
재시작: systemctl restart postgresql-17
로그: journalctl -u postgresql-17 -f
⚠️ 운영 환경에서는 0.0.0.0/0 대신 허용할 IP 대역으로 제한하세요.
