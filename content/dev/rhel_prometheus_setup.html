Node Exporter 설치
1
Node Exporter 압축 해제 및 배치

Node Exporter는 Linux 호스트의 CPU·메모리·디스크·네트워크 메트릭을 HTTP endpoint로 노출합니다. 단독 바이너리로 별도 런타임이 필요 없습니다.

# 설치 디렉토리 생성
mkdir -p /opt/node_exporter

# 압축 해제 (최상위 디렉토리 제거)
tar -xzf node_exporter-1.8.1.linux-amd64.tar.gz \
    -C /opt/node_exporter/ --strip-components=1

# 바이너리 확인
ls /opt/node_exporter/
--strip-components=1 — 최상위 디렉토리를 제거하고 바이너리를 /opt/node_exporter/ 바로 아래에 배치합니다.
2
전용 유저 생성 및 실행

보안 원칙상 nologin 쉘로 설정된 시스템 전용 계정으로 최소 권한 실행합니다.

유저 생성 
useradd -r -s /sbin/nologin node_exporter
chown -R node_exporter:node_exporter /opt/node_exporter
방법 A — nohup (컨테이너·테스트 환경) 
nohup /opt/node_exporter/node_exporter \
    --web.listen-address=":9100" \
    > /var/log/node_exporter.log 2>&1 &

echo "Node Exporter PID: $!"
curl -s http://localhost:9100/metrics | head -5
방법 B — systemd (실 RHEL 서버) 
cat > /etc/systemd/system/node_exporter.service <<'EOF'
[Unit]
Description=Node Exporter
After=network.target

[Service]
User=node_exporter
ExecStart=/opt/node_exporter/node_exporter \
    --web.listen-address=":9100"
Restart=always

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable --now node_exporter
systemctl status node_exporter
✅ 정상 확인 — curl http://localhost:9100/metrics 실행 시 # HELP ... 형태의 메트릭 데이터가 출력되면 정상입니다.
Prometheus 설치
3
Prometheus 압축 해제 및 배치
mkdir -p /opt/prometheus

tar -xzf prometheus-2.53.0.linux-amd64.tar.gz \
    -C /opt/prometheus/ --strip-components=1

mkdir -p /opt/prometheus/data

ls /opt/prometheus/
주요 파일
prometheus — 서버 바이너리  |  promtool — 설정 검증 도구  |  prometheus.yml — 기본 설정 파일
4
전용 유저 생성 및 prometheus.yml 설정
유저 생성 
useradd -r -s /sbin/nologin prometheus
chown -R prometheus:prometheus /opt/prometheus
/opt/prometheus/prometheus.yml 
cat > /opt/prometheus/prometheus.yml <<'EOF'
global:
  scrape_interval:     15s
  evaluation_interval: 15s

scrape_configs:

  # Prometheus 자기 자신 모니터링
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  # Node Exporter — 시스템 메트릭
  - job_name: 'node'
    static_configs:
      - targets: ['localhost:9100']
EOF

# 설정 파일 문법 검증
/opt/prometheus/promtool check config /opt/prometheus/prometheus.yml
✅ 검증 성공 — SUCCESS: /opt/prometheus/prometheus.yml is valid prometheus config file syntax
5
Prometheus 실행 및 전체 동작 확인
방법 A — nohup (컨테이너·테스트 환경) 
nohup /opt/prometheus/prometheus \
    --config.file=/opt/prometheus/prometheus.yml \
    --storage.tsdb.path=/opt/prometheus/data \
    --web.listen-address=":9090" \
    --storage.tsdb.retention.time=15d \
    > /var/log/prometheus.log 2>&1 &

echo "Prometheus PID: $!"
sleep 2 && curl -s http://localhost:9090/-/healthy
방법 B — systemd (실 RHEL 서버) 
cat > /etc/systemd/system/prometheus.service <<'EOF'
[Unit]
Description=Prometheus Monitoring
After=network.target

[Service]
User=prometheus
ExecStart=/opt/prometheus/prometheus \
    --config.file=/opt/prometheus/prometheus.yml \
    --storage.tsdb.path=/opt/prometheus/data \
    --web.listen-address=":9090" \
    --storage.tsdb.retention.time=15d
Restart=always

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable --now prometheus
systemctl status prometheus
최종 확인 
# 프로세스 확인
ps aux | grep -E 'prometheus|node_exporter'

# 포트 리스닝 확인
ss -tlnp | grep -E '9090|9100'

# Prometheus 헬스체크
curl -s http://localhost:9090/-/healthy

# 타겟 상태 API 확인
curl -s http://localhost:9090/api/v1/targets | grep -o '"health":"[^"]*"'
✅ 정상 확인 — Prometheus Server is Healthy.
브라우저: http://서버IP:9090 → Prometheus Web UI (Status → Targets에서 Node Exporter 상태 확인)

Prometheus 중지: pkill prometheus / 로그: tail -f /var/log/prometheus.log
